These rules try to enforce the google style guide

[*]  Semicolons are always required.
     https://eslint.org/docs/rules/semi

[*]  Identation is set to two spaces.
     https://eslint.org/docs/rules/indent

[*]  Strict equality is enforced via ===. There is no use of ==. 
     https://eslint.org/docs/rules/eqeqeq

[*]  Maximum block nesting is set to 5 levels.
     https://eslint.org/docs/rules/max-depth

[*]  In a function defenition, there is no space between function and the parantheses.
     https://eslint.org/docs/rules/space-before-function-paren

[*]  Template literal variables must have a space between the variable and the brackets - ${ test }.
     https://eslint.org/docs/rules/template-curly-spacing

[*]  Use single quotes only
     https://eslint.org/docs/rules/quotes
     https://google.github.io/styleguide/jsguide.html#features-strings-use-single-quotes

[*]  Require curly braces for all control statements.
     https://eslint.org/docs/rules/curly

[*]  Brace style for control statements is the 'one true brace style'.
     https://eslint.org/docs/rules/brace-style

[*]  Always ensure a space before blocks.  Note other rules might override this.
     https://eslint.org/docs/rules/space-before-blocks

[*]  No trailing white space
     https://eslint.org/docs/rules/no-trailing-spaces

[*]  MORE
     https://eslint.org/docs/rules/jsx-quotes
